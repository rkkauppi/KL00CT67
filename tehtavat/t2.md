# KL00CT67 Web-ohjelmoinnin perusteet
## Tehtävä 2
### Portfolion etusivun julkaisu ja muokkaus

a) Kirjaudu GitHubiin (tai luo itsellesi GitHub-tunnus, jos sinulla ei vielä
ole sellaista) ja luo itsellesi julkinen projekti (repository, repo)
Web-ohjelmointia varten. Tee projektiin `README.md`-tiedosto, jonka sisältönä
on tieto projektin tarkoituksesta. Kopioi aiemmin luomasi `index.html`
projektiin.

b) Käy julkaisemassa projektisi asetuksen kohdasta `Pages`. Selvitä, mikä on
`index.html`-sivusi osoite ja varmista, että sivu näkyy selaimella ko.
osoitteessa.

c) Avaa GitHub-projektisi Visual Studio Codessa, tee `index.html`-tiedostoon
muutos, vahvista se ja varmista, että muutos meni GitHubiin. Tarkista lisäksi,
että muutos näkyy, kun avaat `index.html`-sivusi selaimessa.

### Ohjeita

Tulevat tehtävät tehdään tähän projektiin (repoon). Huomioithan, että projekti
on julkinen eli sisällön näkee, jos tietää linkin. Älä siis laita projektiin henkilökohtaisia tietojasi tai sisältöä, johon sinulla ei ole oikeuksia.

Kohdassa c) on kaksi vaihtoehtoa, projektin avaaminen etänä, `Command Palette
(Ctrl + Shift + P)`, `Remote: Show Remote Menu (CTRL + Alt + O)` tai projektin kloonaaminen `Source Control`-näkymässä `Clone Repository`. Kokeile molempia
ja huomioi seuraavat näkökohdat:
- Avaaminen etänä ei edellytä Git-asennusta käytettävälle koneelle, mutta se
ei luo paikallista kopiota projektista. Voit käyttää tätä väliaikaisesti,
jos et ole vielä asentanut Git-versionhallintaa koneellesi.
- Kloonaaminen edellyttää Git-asennusta sekä GitHub-käyttäjänimen
sekä -sähköpostin asettamista. Tämä on jatkossa suositeltava tapa.

Git-asennuspaketti, ks. [git
Install](https://git-scm.com/install/windows)
ja käyttäjänimen ja sähköpostin asetus, ks. [Fist-Time Git Setup
](https://git-scm.com/book/en/v2/Getting-Started-First-Time-Git-Setup).
Käyttäjänimi ja sähköposti lyhyesti - esimerkiksi terminaalissa seuraavat 
komennot projektin ollessa auki VS Codessa:
```
git config user.name "John Doe"
git config user.email johndoe@example.com
```

(Ylläoleva asettaa käyttäjänimen ja sähköpostin ko. projektille. Seuraavat
versiot asettavat vastaavat globaalisti:
```
git config --global user.name "John Doe"
git config --global user.email johndoe@example.com
```
)
